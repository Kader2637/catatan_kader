Modul 1: Pengenalan Vue 3 & Composition API

Modul 01

"Membangun Antarmuka Interaktif yang Elegan"

Diperbarui Terakhir
45 Menit Eksekusi

Vue 3 memperkenalkan Composition API melalui <script setup>. Sintaks ini sangat bersih, minim *boilerplate*, dan membuat penulisan logika JavaScript menjadi lebih natural (sangat mirip dengan *Vanilla JS* biasa) namun tetap *reactive*.

Dasar Sintaks Vue 3

Reactivity & Event Handling

01 Struktur Single File Component (SFC)

Dalam Vue, satu file .vue memiliki 3 bagian utama: <script> untuk logika, <template> untuk HTML, dan <style> untuk CSS.

<script setup>
// Di sini tempat logika Javascript
const namaSistem = "Catatan Kader APP";
</script>

<template>
  <!-- Gunakan kurung kurawal ganda untuk menampilkan variabel -->
  <h1>Selamat Datang di {{ namaSistem }}</h1>
</template>

<style scoped>
/* CSS khusus untuk komponen ini saja */
h1 { color: #42b883; }
</style>

02 Reactivity dengan `ref`

Agar variabel HTML bisa berubah secara otomatis saat datanya berubah, kita membungkus nilai tersebut dengan ref().

<script setup>
import { ref } from 'vue';

// Membuat variabel reaktif
const angka = ref(0);

// Cara mengubah nilainya di dalam script HARUS menggunakan .value
const tambah = () => {
  angka.value++;
};
</script>

<template>
  <div>
    <!-- Di template, TIDAK PERLU pakai .value -->
    <p>Nilai saat ini: {{ angka }}</p>
    <button @click="tambah">Tambah Angka</button>
  </div>
</template>
Kader Note (Composition API):

Dulu di Vue 2 kita menggunakan *Options API* (data, methods, computed). Dengan <script setup> di Vue 3, kodenya jauh lebih sedikit, tidak perlu lagi menggunakan kata kunci this, dan sangat mudah untuk memecah logika yang panjang ke file terpisah (Composables).
